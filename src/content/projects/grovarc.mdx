## 문제 정의

개발자가 성장하고 있다는 걸 스스로 실감하기 어려운 이유:
- 매일 작업하지만 무엇을 했는지, 어떻게 성장했는지 정리가 어려움
- 회고를 쓰고 싶어도 막막하고 시간이 오래 걸림
- 학습 패턴과 개선점을 객관적으로 파악할 수단이 없음

**목표**: 개발자가 작업 로그를 기록하면 AI Agent가 패턴을 분석해 회고 초안, 성장 리포트, 학습 로드맵을 자동으로 제공하는 서비스

## 역할과 범위

**역할**: 솔로 풀스택 개발자 (AI-assisted 개발)
**범위**:
- 프론트엔드: Next.js 작업 로그 기록 및 대시보드
- 백엔드: Kotlin Spring Boot REST API
- AI 서버: Python FastAPI + LangGraph AI Agent
- MCP 서버: Claude와 직접 인터랙션하는 TypeScript MCP 서버
- 인프라: Kubernetes, Terraform, Prometheus, Grafana

## 핵심 설계

### 1. LangGraph AI Agent 파이프라인

작업 로그를 수신하면 LangGraph로 구성된 Agent가 단계적으로 분석:

```python
# LangGraph 기반 회고 생성 Agent
from langgraph.graph import StateGraph, END
from typing import TypedDict

class AgentState(TypedDict):
    logs: list[str]
    patterns: dict
    draft: str

def analyze_patterns(state: AgentState) -> AgentState:
    # 작업 로그에서 패턴 추출 (반복 주제, 학습 흐름 등)
    patterns = extract_patterns(state["logs"])
    return {**state, "patterns": patterns}

def generate_retrospective(state: AgentState) -> AgentState:
    # LLaMA 3 + RAG로 회고 초안 생성
    prompt = build_prompt(state["patterns"])
    draft = llama_model.generate(prompt)
    return {**state, "draft": draft}

# Graph 구성
graph = StateGraph(AgentState)
graph.add_node("analyze", analyze_patterns)
graph.add_node("generate", generate_retrospective)
graph.add_edge("analyze", "generate")
graph.add_edge("generate", END)
graph.set_entry_point("analyze")

retrospective_agent = graph.compile()
```

### 2. Kotlin Spring Boot — 로그 API + Kafka 발행

```kotlin
@Service
class WorkLogService(
    private val workLogRepository: WorkLogRepository,
    private val kafkaTemplate: KafkaTemplate<String, WorkLogEvent>
) {
    fun createLog(userId: Long, request: WorkLogRequest): WorkLog {
        val log = workLogRepository.save(
            WorkLog(userId = userId, content = request.content, date = LocalDate.now())
        )

        // AI 분석 트리거를 Kafka로 발행
        kafkaTemplate.send(
            "work-log-created",
            WorkLogEvent(logId = log.id, userId = userId, content = log.content)
        )

        return log
    }
}
```

## 기술 선택 이유

### Kotlin (BE)
- JVM 생태계와 Spring Boot 호환성 유지하면서 간결한 코드
- 코루틴 기반 비동기 처리로 AI 서버 응답 대기 시 효율적

### Python + FastAPI (AI)
- LangGraph, LLaMA 3 등 AI/ML 생태계가 Python에 집중
- FastAPI의 async 지원으로 LLM 추론 대기 처리

### LangGraph
- 단순 LLM 호출이 아닌 복잡한 분석 흐름(패턴 추출 → 회고 생성 → 로드맵 제안)을 그래프로 표현
- 조건부 분기, 루프, 상태 관리를 명확하게 모델링

### MCP SDK
- Claude Desktop과 직접 연동하여 사용자가 Claude와 대화하며 자신의 성장 데이터를 조회/분석 가능
